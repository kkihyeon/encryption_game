## 1. 출제자 암호문 및 키 입력

**요구사항**: 암호화 종류 선택 후, 출제자가 직접 암호화될 암호문과 **키**를 입력해야 한다.

**현재 상태**: ✅ 적용됨 *(2026-04-15 구현)*

- 암호화 방식 선택 시 해당 방식에 맞는 키 입력 필드(`<input type="number">`)가 동적으로 생성됨 (`renderKeyInput()`)
- 방식별 키 기본값 및 범위: 카이사르(shift, 1–25), 스키테일(레일, 2–10), 키 순서(모드값, 1–20), 애너그램(키 없음)
- 암호화 함수 (`caesarEnc`, `keyEnc`, `scytaleEnc`)가 키 파라미터를 받도록 수정됨
- 키값은 `gameState.currentKeys` 필드에 저장되어 전 플레이어에게 전파됨
- 출제 제출(`enc_submit`) 메시지에 `keys` 객체 포함

---

## 2. 틀릴 시 점수 처리 (감점 / 음수 점수 방지)

**요구사항**: 틀리면 점수 X 또는 감점. 음수 점수는 없다.

**현재 상태**: ✅ 적용됨

- `const pts = success ? (100 + timeBonus) : 0;` — 틀리면 0점 부여 (점수 X)
- 음수 점수가 발생하는 로직 없음
- 단, "감점" 옵션은 구현되어 있지 않음 (0점 처리만 존재)

**비고**: "점수 X 또는 감점" 요구사항에서 현재는 "점수 X(0점)" 방식을 택하고 있음. 감점 방식이 필요하다면 `Math.max(0, score - penalty)` 패턴으로 별도 구현 필요.

---

## 3. 정답자에게 출제자가 사용한 키값 공개

**요구사항**: 문제를 맞춘 사람들은 출제자가 사용한 키값을 알 수 있어야 한다.

**현재 상태**: ✅ 적용됨 *(2026-04-15 구현)*

- `gameState.currentKeys` 필드에 방식별 키값이 저장됨
- `round_end` 페이즈에서 `renderCenter()`가 정답자(`guessResults[myId]?.correct === true`) 또는 출제자(`iAmEncoder`)에게만 키값 힌트 chip 표시
- 표시 형태: `🔑 카이사르: 키=3 → 스키테일: 키=4` 형태로 렌더링

---

## 4. 암호화 방식 2개 사용 시 종류(순서 X)를 힌트로 제공

**요구사항**: 암호화 방식을 2개 사용하면 제출자들에게 순서가 아닌 종류만 알려주어야 한다.

**현재 상태**: ✅ 적용됨

- `data.methods.length >= 2` 조건 하에 `[...data.methods].sort(() => Math.random() - 0.5)` 로 순서를 셔플하여 `clueSet` 생성
- `renderCenter()`에서 `gameState.currentMethods.length >= 2`일 때만 힌트 chip 표시
- 순서 정보 없이 사용된 암호화 종류만 시각적으로 노출됨

---

## 5. 암호화 방식 1개 사용 시 힌트 비공개

**요구사항**: 암호화 방식을 1개만 사용했을 때는 해독자들에게 암호화 방식을 알려주지 않아야 한다.

**현재 상태**: ✅ 적용됨 *(2026-04-15 구현)*

- `handleData` (`enc_submit`)에서 `methods.length >= 2`일 때만 `clueSet`을 채우고, 1개일 때는 `clueSet = []`
- `renderCenter()` 힌트 렌더링 블록에 `if (gameState.currentMethods.length >= 2)` 조건 추가
- 해독 UI(`renderGuessUI`)에서 1개 방식 시 순서 선택 버튼 미렌더링; 라벨도 "원본 메시지 입력"으로 변경
- `submitGuess()`에서 1개 방식 시 순서 체크 스킵 (`isSingle` 분기)
- 호스트의 `guess_submit` 처리에서도 `isSingleMethod ? true : orderOK` 로 1개 방식 시 순서 검증 면제

## 7-1. 게임 가이드 탭 콘텐츠

**요구사항**: 게임 가이드를 누르면 실제 게임 화면을 일시적으로 보여주며 게임 내 기능들을 소개한다. X 버튼을 누르면 메인 화면으로 돌아온다.

**현재 상태**: ✅ 적용됨 *(2026-04-15 구현)*

- `#gpanel-game` 패널에 게임 화면 구성요소별 설명 callout 카드 구현:
  - 상단 헤더(페이즈/타이머/라운드), 좌측 패널(참가자/순위), 중앙 영역(캔버스/암호문), 하단 액션 영역
  - 게임 흐름(출제→해독→결과공개), 점수 규칙, 그리기/메모장 모드 설명 포함
- X 버튼으로 오버레이 닫고 로비 화면 복귀

**비고**: 실제 게임 화면 렌더링 대신 설명형 callout 카드 방식으로 구현됨. 실제 게임 UI 미리보기가 필요하다면 `#game-ui`를 preview 모드로 표시하는 추가 작업 필요.

## 잔여 개선 고려사항 (필수 아님)

- **감점 옵션**: 현재는 오답 시 0점 처리. 감점 방식 원하면 `Math.max(0, score - penalty)` 추가 필요
- **게임 가이드 실제 UI 미리보기**: 현재 텍스트 callout 방식; 실제 게임 화면을 비활성 상태로 보여주는 방식 원하면 별도 구현 필요
- **메모장 공유**: 현재 개인 메모 전용. 공유 메모장 원하면 PeerJS 통신에 `memo_update` 이벤트 추가 필요

## 1. 출제자 암호문 및 키 입력

**요구사항**: 암호화 종류 선택 후, 출제자가 직접 암호화될 암호문과 **키**를 입력해야 한다.

**현재 상태**: ⚠️ 부분 적용

- 암호화 방식 선택 버튼(`method-selector`)과 원본 메시지 입력(`raw-input`) 필드는 존재함
- 자동 암호화 결과가 채워지거나 직접 결과를 입력할 수 있음
- **키(key) 입력 필드가 없음**: 각 암호화 방식에 대한 키값을 출제자가 별도로 입력하는 UI 및 로직이 구현되어 있지 않음
- 현재 암호화 함수(`caesarEnc`, `keyEnc`, `scytaleEnc` 등)는 하드코딩된 값 사용 (고정 shift=3, 고정 rail=3, 고정 i%5)

**구현 필요 항목**:
- 각 암호화 방식 선택 시 해당 방식에 맞는 키 입력 필드 동적 생성
- 키값을 `gameState`에 포함시켜 전파
- 암호화 함수에 키 파라미터 전달하도록 수정
- 예시 키 입력 형태: 카이사르 → 숫자(shift), 스키테일 → 숫자(rail 수), 키 순서 → 숫자/문자열, 애너그램 → 별도 키 없음 또는 사용자 정의

## 5. 암호화 방식 1개 사용 시 힌트 비공개

**요구사항**: 암호화 방식을 1개만 사용했을 때는 해독자들에게 암호화 방식을 알려주지 않아야 한다.

**현재 상태**: ❌ 미적용

- `renderCenter()` 함수에서 `gameState.clueSet`를 항상 순회하며 힌트 chip을 표시함
- 1개든 2개든 사용된 모든 방식이 힌트로 노출됨

**구현 필요 항목**:
- `renderCenter()` 내 힌트 렌더링 부분에 조건 추가:
  ```js
  if (gameState.currentMethods.length >= 2) {
    gameState.clueSet.forEach(mId => { /* hint chip 생성 */ });
  }
  // 1개일 때는 힌트 chip을 렌더링하지 않음
  ```
- 힌트 chip이 없을 때 추측 UI에서 사용자가 방식을 직접 선택하게 하는 로직은 이미 존재함 (`renderGuessUI()`에서 `clueSet` 기반으로 버튼 생성)
- 단, `clueSet`이 1개인 경우 추측 UI의 방식 선택 버튼도 숨겨야 하는지 별도 확인 필요

## 6. 메모장 모드 (텍스트 입력) / 그리기 모드 전환

**요구사항**: 메모장 모드(텍스트 입력)와 그리기 모드가 있어야 한다.

**현재 상태**: ⚠️ 부분 적용

- 그리기 모드: ✅ 구현됨 — `setMode('draw')`, 캔버스 드로잉, 색상 선택, 지우기 모두 동작
- 메모장 모드: ❌ 미구현
  - 툴바에 "메모장"이라는 텍스트 레이블만 존재 (`<span style="font-size:11px;color:var(--text3);letter-spacing:2px;font-weight:700">메모장</span>`)
  - 텍스트 입력 영역이 없음
  - 메모장/그리기 전환 버튼이 없음

**구현 필요 항목**:
- 툴바에 "메모장 모드" / "그리기 모드" 전환 버튼 추가
- 메모장 모드 활성화 시 캔버스 숨기고 `<textarea>` 영역 표시
- 그리기 모드 활성화 시 텍스트 영역 숨기고 캔버스 표시
- 텍스트 내용을 다른 플레이어에게 공유할지 여부 결정 필요 (캔버스처럼 실시간 동기화 or 단순 개인 메모)

## 7. 게임 가이드 탭 (로비 화면)

**요구사항**: 게임 시작 전 로비에 "게임 가이드" 탭이 있어야 하며, 게임 가이드와 암호화 가이드를 선택할 수 있어야 한다.

**현재 상태**: ❌ 미적용

- 현재 로비 화면(`#lobby`)에는 "방 만들기" / "방 참가하기" 탭만 존재
- 게임 가이드 관련 UI, 탭, 모달, 오버레이 등 어떠한 형태도 없음

**구현 필요 항목**:
- 로비 카드(`lobby-card`) 상단 또는 별도 진입점에 "게임 가이드" 버튼 추가  
- 게임 가이드 오버레이/모달 HTML 구조 생성
- 내부에 "게임 가이드" / "암호화 가이드" 탭 전환 UI 구현
- 각 탭 콘텐츠 구현 (아래 7-1, 7-2 참조)

---

## 7-1. 게임 가이드 탭 콘텐츠

**요구사항**: 게임 가이드를 누르면 실제 게임 화면을 일시적으로 보여주며 게임 내 기능들을 소개한다. X 버튼을 누르면 메인 화면으로 돌아온다.

**현재 상태**: ❌ 미적용

**구현 필요 항목**:
- 게임 UI(`#game-ui`)를 오버레이 위에 렌더링하거나, 게임 화면 스크린샷/시뮬레이션을 보여주는 방식으로 구현
- 각 UI 요소(헤더 타이머, 플레이어 패널, 암호문 표시 영역, 액션 영역, 툴바)에 설명 레이블(툴팁 또는 callout) 추가
- X 버튼 클릭 시 가이드 오버레이 닫고 로비 화면 복귀
- 권장 구현 방식: 포지션 `fixed` 오버레이 + 투명 배경으로 실제 `#game-ui`를 비활성 상태로 보여주거나, 별도 이미지/HTML 미리보기 사용

---

## 7-2. 암호화 가이드 탭 콘텐츠

**요구사항**: 각 암호화 방식이 어떤 식으로 이루어지는지에 대한 설명을 볼 수 있어야 한다.

**현재 상태**: ❌ 미적용

**구현 필요 항목**:
- 4가지 암호화 방식에 대한 설명 콘텐츠 작성 필요:

| 방식 | 설명 포함 내용 |
|------|---------------|
| 키 순서 (KEY) | 각 문자의 ASCII 코드에 `(index % 5) + 1`을 더함. 키: 정수(모드값) |
| 애너그램 (ANA) | 문자열을 역순으로 뒤집음. 키: 없음(또는 사용자 정의 치환 표) |
| 카이사르 (CAE) | 알파벳을 n칸씩 시프트. 기본 shift=3. 키: 시프트 숫자 |
| 스키테일 (SKY) | rail 수 단위로 행 분할 후 열 순서로 재조합. 기본 rail=3. 키: rail 수 |

- 예시 입력/출력 표시 (인터랙티브 또는 정적)
- 현재 암호화 함수 동작 방식을 UI에 반영해야 하며, 키 입력 기능(요구사항 1) 구현 후 설명도 업데이트 필요

## 구현 우선순위 권장

1. **[높음] 요구사항 5** — `renderCenter()` 한 줄 조건 추가로 해결 가능. 사이드 이펙트 없음
